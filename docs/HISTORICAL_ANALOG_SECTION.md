# 《历史情景对照与普通人理解指南》模块说明

## 修改方案概述

- **目标**：在报告尾部、免责声明之前，新增一个面向普通读者的「人话翻译层」，不修改任何评分逻辑。
- **内容**：历史情景静态库 + 当前信号匹配 + 历史参照表 +「当前距离更严重确认还差什么」+ 普通人理解 3–5 条。
- **原则**：历史参照框架，非预测；避免假精确、过强交易措辞、危言耸听；部分匹配、标注置信度。

## 修改文件清单

| 文件 | 变更 |
|------|------|
| `historical_analogs.py` | 已有：`HISTORICAL_ANALOGS` 静态库、`match_historical_analogs()`、`get_confidence_from_matches()`；顶部增加维护约定说明。 |
| `crisis_monitor_v2.py` | 已有：`generate_historical_analog_section(report_data)`；补全 `report_signals`（de_dollarization、k_shaped、japan_contagion）；信号缺失时强制置信度 LOW；Part B 表格文案微调；Part C 不再截断「还需要达到什么」。 |
| `docs/HISTORICAL_ANALOG_SECTION.md` | 本说明与示例。 |

## 新增区块在报告中的位置

- 插入在 **「⚠️ 免责声明」** 之前。
- 若存在免责声明，则在其前插入整段《历史情景对照》+《普通人如何理解今天的风险》；否则追加到报告末尾。

## 新增区块示例（Markdown）

```markdown
## 📚 历史情景对照：如果历史重演，普通人通常会先看到什么？

> ⚠️ 以下内容为历史参照，不是预测。当前匹配只表示「更像哪类历史阶段」，不表示未来一定重演。

当前匹配置信度：**MEDIUM**

**主要匹配情景**：**2018 年流动性收紧 / 企业融资压力**（匹配度 0.7，partial）

**历史参照表**

| 维度 | 历史上更常见的情况 | 这对普通人的意思 |
|------|-------------------|------------------|
| 这次更像哪类阶段 | 2018 年流动性收紧 / 企业融资压力 | 这类阶段常先是边缘资产变弱，不一定立刻变成全面危机。 |
| 常见演化时间窗 | 2–8 周 | 历史上反应多为周至月，不是单日见分晓 |
| 常见先受影响资产 | 先是高风险信用资产与高弹性股票承压，随后决定是否扩散 | 通常先波动的是风险偏好高的资产，再扩散 |
| 更严重升级通常需要什么 | 若高收益利差继续走阔并伴随金融压力抬升 | 需多条风险腿同时恶化，单腿不足为据 |
| 若历史重演，常见市场表现 | 先是高风险信用资产与高弹性股票承压，随后决定是否扩散 | 这类阶段常先是边缘资产变弱，不一定立刻变成全面危机。 |

**当前距离「更严重确认」还差什么**

| 信号 | 当前状态 | 还需要达到什么 | 历史上通常意味着什么 |
|------|----------|----------------|------------------------|
| Private Credit | WATCH | HY OAS 明显走阔、金融压力继续上升 | 若升级，未来数周风险资产更易波动放大 |
| Geopolitics | — | 地缘/通胀链多腿同步确认 | 只有升级后才更接近系统性风险 |
| Resonance | OFF | 多条风险腿同步确认 | 只有升级后才更接近链式反应 |
| Warsh / Liquidity | （见报告正文） | 流动性 plumbing 压力抬升 | 银行间与货币市场压力 |
| Overall risk | 早期压力 | 多腿共振或 WATCH→ALERT | 连续升级比单日亮灯更重要 |

## 🧭 普通人如何理解今天的风险

- 信号亮了，不等于明天就崩盘；连续多日维持或升级，比单日闪现更重要。
- 单腿预警更像天气变化，多腿共振才更像风暴形成。
- 股票、债券、商品的反应速度不同，先波动的常是风险偏好高的资产。
- 真正需要提高防守的，是 WATCH 变 ALERT、或 Resonance 升级，不是任何一次小亮灯。
- 单日亮灯更像天气预报，连续升级更像季节变化。
```

## 静态字典是否拆到单独文件

- **当前**：`HISTORICAL_ANALOGS` 放在 `historical_analogs.py` 中，与匹配逻辑同文件，便于 code review 时一起看。
- **建议**：若后续希望由非开发在配置中增删情景，可迁至 `config/historical_analogs.yaml`（结构同上），在 `historical_analogs.py` 启动时加载并 fallback 到代码内默认 dict；当前保持现状即可。

## 约束（已遵守）

- 不修改 Base / Event-X / Structural / Resonance 评分逻辑。
- 不改变现有报告结论。
- 不使用外部在线数据源。
- 文案克制，无恐吓式表达，不把历史类比写成「未来保证」。
